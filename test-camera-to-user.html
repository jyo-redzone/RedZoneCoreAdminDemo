Camera to User Navigation Test

This test verifies the specific bug where camera names linger in breadcrumbs when navigating to users.

Test Steps

Step 1: Navigate to Camera

Action: Navigate to "Pitches" → "Main Stadium" → Click any camera

Expected BC: ["Home", "Pitches", "Main Stadium"] (Depth: 3)

Expected Path: {pitchId: "pitch-1"}

Expected Kind: cameras

Expected: Details panel opens

Step 2: Navigate to User (THE BUG TEST)

Action: Navigate to "Palermo" → "Football" → "Men's First Team" → Click any user

Expected BC: ["Home", "Palermo", "Football", "Men's First Team"] (Depth: 4)

Expected Path: {clubId: "club-1", sportId: "sport-1", teamId: "team-1"}

Expected Kind: users

Expected: NO camera/pitch names in breadcrumb

Bug Check: Breadcrumb should NOT contain "Main Stadium" or "Pitches"

Step 3: Navigate to Different Club

Action: Navigate to "Lommel" → "Football" → "First Team"

Expected BC: ["Home", "Lommel", "Football", "First Team"] (Depth: 4)

Expected Path: {clubId: "club-2", sportId: "sport-4", teamId: "team-9"}

Expected Kind: users

Expected: NO Palermo or pitch names in breadcrumb

Step 4: Navigate Back to Pitches

Action: Navigate to "Pitches" → "Training Ground A"

Expected BC: ["Home", "Pitches", "Training Ground A"] (Depth: 3)

Expected Path: {pitchId: "pitch-2"}

Expected Kind: cameras

Expected: NO club/sport/team names in breadcrumb
